## Theme Customization

The theme customization system allows SUPER_ADMIN users to completely rebrand the CONIITI website with custom colors, logos, banners, and content across all pages.

<Warning>
Theme customization is **only available to SUPER_ADMIN** users. Other admin roles cannot access these settings.
</Warning>

## Accessing Theme Settings

<Steps>
  <Step title="Navigate to Configuración">
    Click **⚙️ Configuración Página** in the admin sidebar.
  </Step>

  <Step title="Select Settings Category">
    Use the sub-navigation tabs to access different configuration areas:
    - ⚙️ **General** - Core site settings
    - 🏠 **Inicio** - Home page customization
    - 👥 **Invitados** - Speakers page settings
    - 📅 **Agenda** - Schedule page configuration
    - ℹ️ **Acerca De** - About page content
    - ✉️ **Contacto** - Contact information
    - 🏛️ **Aforos y Sedes** - Venue management
  </Step>
</Steps>

## General Settings

### Event Name

Set the primary event name displayed throughout the site:

```tsx
<div className="form-group">

```

### Theme Selection

Choose from predefined country themes:

```typescript
// Admin.tsx:1047-1063
<select
    defaultValue={localStorage.getItem("site_theme") || "default"}
    onChange={(e) => {
        const theme = e.target.value;
        localStorage.setItem("site_theme", theme);
        document.body.className = theme === "default" ? "" : `theme-${theme}`;
        dispatchUpdate();
    }}
    className="theme-select"
>
    <option value="default">Estándar (Universidad Católica)</option>
    <option value="colombia">Colombia (Amarillo, Azul y Rojo)</option>
    <option value="italy">Italia (Verde, Blanco y Rojo)</option>
    <option value="mexico">México (Verde y Rojo)</option>
</select>
```

<Info>
Theme selection applies CSS class to body element, triggering predefined color schemes. Custom themes can be added via CSS.
</Info>

### Banner Image

Upload a custom header banner:

<Steps>
  <Step title="View Current Banner">
    The current banner displays in a preview box (100px height).
  </Step>

  <Step title="Upload New Banner">
    Click the file input to select an image:
    
    ```typescript
    // Admin.tsx:1073-1088
    <input
        type="file"
        accept="image/*"
        onChange={(e) => {
            const file = e.target.files?.[0];
            if (file) {
                if (file.size > 1500000) { 
                    toast.error("La imagen es muy pesada (máx 1.5MB)."); 
                    return; 
                }
                const reader = new FileReader();
                reader.onloadend = () => {
                    if (reader.result) {
                        try { 
                            localStorage.setItem("site_banner", reader.result as string); 
                            dispatchUpdate(); 
                        }
                        catch { toast.error("La imagen sigue siendo muy grande."); }
                    }
                };
                reader.readAsDataURL(file);
            }
        }}
        className="file-input"
    />
    ```
    
    <Warning>
    Banner images must be under **1.5MB**. Larger images will be rejected with an error message.
    </Warning>
  </Step>

  <Step title="Restore Default">
    Click **Restaurar Original** to revert to the default banner.
  </Step>
</Steps>

Upload custom logos for both university and event branding:

```tsx
// Admin.tsx:1098-1133
<div className="logos-edit-grid">

        <input type="file" accept="image/*" onChange={(e) => {
            const file = e.target.files?.[0];
            if (file) {
                if (file.size > 800000) { 
                    alert("El logo es muy grande (máx 800KB)."); 
                    return; 
                }
                const reader = new FileReader();
                reader.onloadend = () => { 
                    if (reader.result) { 
                        localStorage.setItem("site_logo_uni", reader.result as string);

                    }

                onClick={() => { localStorage.removeItem("site_logo_uni"); dispatchUpdate(); }}>
            Restaurar Original
        </button>
    </div>
    
    {/* Event Logo (similar structure) */}
</div>
```

<Note>
Logos are limited to **800KB** each. Both logos display in the header navigation across all pages.
</Note>

### Real-time Updates

Color changes dispatch update events to refresh the UI:

```typescript
// Admin.tsx:44-54
const dispatchUpdate = () => {
    window.dispatchEvent(new Event('site-config-updated'));
    setBannerPreview(localStorage.getItem("site_banner") || "/banner-header.png");
    setLogoUniPreview(localStorage.getItem("site_logo_uni") || "/ucatolica-logo.png");
    setLogoEventPreview(localStorage.getItem("site_logo_evento") || "/logo-coniiti.png");
    setBgColor(localStorage.getItem("custom_bg_color") || "#ffffff");
    setTextColor(localStorage.getItem("custom_text_color") || "#1b1a1a");
    setHeaderColor(localStorage.getItem("custom_header_bg") || "#1f2a44");
    setPrimaryColor(localStorage.getItem("custom_primary_color") || "#2563EB");
    setSecondaryColor(localStorage.getItem("custom_secondary_color") || "#1E293B");
};
```

<Tip>
Changes update immediately across all pages listening for the `site-config-updated` event. No page reload required.
</Tip>

### Reset Customization

Clear all custom styling and revert to defaults:

```typescript
// Admin.tsx:1168-1177
<button type="button" className="btn-logout" 
    style={{ marginLeft: '1rem', background: '#e74c3c' }}
    onClick={() => {
        if (confirm("¿Seguro que quieres borrar toda la personalización?")) {
            ["site_banner", "site_logo_uni", "site_logo_evento", 
             "custom_bg_color", "custom_text_color", "custom_header_bg", 
             "custom_primary_color", "custom_secondary_color"]
                .forEach(key => localStorage.removeItem(key));
            dispatchUpdate();
        }
    }}>
    Limpiar y Resetear Diseño
</button>
```

<Warning>
Resetting design clears all customizations **permanently**. This action cannot be undone. Always confirm before proceeding.
</Warning>

## Best Practices

<AccordionGroup>
  <Accordion title="Image Optimization">
    - Compress images before upload
    - Use appropriate dimensions: Banner (1920x400px), Logos (200x80px)
    - Convert to WebP or optimized JPEG/PNG
    - Stay well under file size limits to avoid localStorage issues
  </Accordion>

  <Accordion title="Color Accessibility">
    - Ensure sufficient contrast between text and background (WCAG AA minimum 4.5:1)
    - Test color combinations with accessibility tools
    - Avoid pure black (#000000) and pure white (#FFFFFF)
    - Consider color blindness when selecting palettes
  </Accordion>

  <Accordion title="Content Guidelines">
    - Keep hero titles concise (under 50 characters)
    - Write descriptive button text (avoid generic "Click Here")
    - Proofread all custom content before saving
    - Maintain consistent tone across all page descriptions
  </Accordion>

  <Accordion title="Testing">
    - Preview changes in multiple browsers
    - Test responsive layout on mobile devices
    - Verify all images load correctly
    - Check that custom colors work with both light and dark mode (if implemented)
  </Accordion>
</AccordionGroup>

## Troubleshooting

<AccordionGroup>
  <Accordion title="Images not loading after upload">
    This usually indicates the image exceeded localStorage limits. Try:
    - Compressing the image further
    - Using a smaller resolution
    - Converting to a more efficient format (WebP)
    - Checking browser console for quota errors
  </Accordion>

  <Accordion title="Colors not applying">
    Ensure you clicked the Save button and that CSS variables are properly defined. Some browsers cache CSS aggressively - try a hard refresh (Ctrl+Shift+R).
  </Accordion>

  <Accordion title="Changes disappear after refresh">
    Check that JavaScript is enabled and localStorage is not disabled. Some privacy modes or extensions block localStorage access.
  </Accordion>

  <Accordion title="Reset doesn't work">
    If the reset button doesn't restore defaults, manually clear localStorage via browser DevTools:
    ```javascript
    // In browser console
    localStorage.removeItem('site_banner')
    localStorage.removeItem('custom_bg_color')
    // ... etc
    ```
  </Accordion>
</AccordionGroup>
