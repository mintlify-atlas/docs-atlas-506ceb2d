## User Management

The user management system allows SUPER_ADMIN to view all registered conference attendees, analyze registration data, and track user engagement.

<Warning>
User management features are **exclusively available to SUPER_ADMIN** users. ADMIN, CONTENT_MANAGER, and VIEWER roles cannot access this section.
</Warning>

## Accessing User Data

<Steps>
  <Step title="Navigate to Users Tab">
    Click **👤 Usuarios** in the admin sidebar (only visible to SUPER_ADMIN).
  </Step>

  <Step title="View Registered Users">
    All registered users display in a comprehensive table with key information.
  </Step>
</Steps>

## User Data Loading

The system aggregates user data from multiple sources during admin panel initialization:

```typescript
// Admin.tsx:138-220 (fetchAllData function)
const fetchAllData = async () => {
    setIsLoading(true);
    let usuariosData: any[] = [];

    try {
        const { getUsuarios } = await import("../services/api");
        
        // Fetch from API
        try { 
            usuariosData = await getUsuarios(); 
        } catch (e) { 
            console.warn("API Usuarios failed"); 
        }

        // Merge with localStorage users
        const localUsers = JSON.parse(localStorage.getItem('usuarios_locales') || '[]');
        const transformedLocal = localUsers.map((u: any) => ({
            id: u.documentNumber || `local-${u.email}`,
            nombre_completo: u.fullName,
            email: u.email,
            rol: u.role,
            carrera: u.career,
            created_at: new Date().toISOString()
        }));

        // Deduplicate by email
        const dbEmails = new Set(usuariosData.map((u: any) => u.email));
        const uniqueLocals = transformedLocal.filter((u: any) => !dbEmails.has(u.email));
        setRegisteredUsers([...usuariosData, ...uniqueLocals]);

    } catch (err) {
        console.error("Error fetching users:", err);
    } finally {
        setIsLoading(false);
    }
}
```

<Info>
The system merges users from:
1. **Supabase Database** (via `/api/usuarios` endpoint)
2. **LocalStorage** (users registered while API was unavailable)
3. Deduplication ensures no duplicate entries by email
</Info>

## User Data Structure

Users retrieved from the API follow this schema:

```typescript
interface Usuario {
    id: string                        // Document number or generated ID
    nombre_completo: string           // Full name
    email: string                     // Email address (unique)
    rol: string                       // User role (ATTENDEE, SPEAKER, etc.)
    carrera: string                   // Academic program
    genero?: string                   // Gender for analytics
    created_at: string                // ISO timestamp
}
```

## API Integration


The backend endpoint returns user profiles from the `usuario` table in Supabase, joining with authentication data.

## User Analytics

SUPER_ADMIN can access detailed user analytics in the **📊 Analíticas** section.

### Gender Distribution

View registration breakdown by gender:

```typescript
// utils/tracker.ts (referenced in Admin.tsx:4)
export const getGenderStats = () => {
    const users = JSON.parse(localStorage.getItem('usuarios_locales') || '[]');
    const counts = users.reduce((acc: any, user: any) => {
        const gender = user.gender || 'No especificado';
        acc[gender] = (acc[gender] || 0) + 1;
        return acc;
    }, {});
    
    return Object.entries(counts).map(([name, value]) => ({ name, value }));
}
```

Display as pie or bar chart:

```tsx
// Admin.tsx:1760-1799
<div className="analytics-card premium-card">
```

## User Roles

The system supports multiple user roles with different access levels:

| Role | Access Level | Typical User |
|------|--------------|-------------|
| **SUPER_ADMIN** | Full system access | System administrators |
| **ADMIN** | Event management | Event coordinators |
| **CONTENT_MANAGER** | Content editing | Content editors |
| **VIEWER** | Read-only | Auditors, stakeholders |
| **ATTENDEE** | Public user | Conference attendees |
| **SPEAKER** | Speaker portal | Conference speakers |

<Info>
Roles are stored in the `user_session` object in localStorage and verified on component mount.
</Info>

## Best Practices

<AccordionGroup>
  <Accordion title="Privacy Compliance">
    - Only collect necessary user information
    - Implement proper data retention policies
    - Provide users with data access/deletion options
    - Comply with GDPR, CCPA, or local data protection laws
    - Secure localStorage data (consider encryption for sensitive info)
  </Accordion>

  <Accordion title="Data Integrity">
    - Validate email formats during registration
    - Prevent duplicate registrations (check by email)
    - Regularly backup user data from API
    - Implement data reconciliation between localStorage and API
    - Log all user management actions for audit trails
  </Accordion>

  <Accordion title="Analytics Usage">
    - Review analytics weekly to identify trends
    - Monitor registration patterns near event date
    - Track popular sessions for venue planning
    - Use gender data for diversity reporting
    - Analyze page views to optimize content placement
  </Accordion>

  <Accordion title="Performance Monitoring">
    - Set performance budgets (e.g., max 3s load time)
    - Identify slow pages from metrics
    - Optimize images if load times are high
    - Monitor resource sizes to prevent bloat
    - Track metrics across different devices/connections
  </Accordion>
</AccordionGroup>

## Troubleshooting

<AccordionGroup>
  <Accordion title="User count mismatch">
    If the user count differs between localStorage and API:
    1. Check for network errors in browser console
    2. Verify API endpoint is accessible
    3. Run data reconciliation by refreshing admin panel
    4. Check for duplicate emails in both sources
  </Accordion>

  <Accordion title="Analytics not updating">
    Analytics rely on the tracker utility. If data seems stale:
    - Verify `utils/tracker.ts` is properly loaded
    - Check that events are being logged to localStorage
    - Clear tracker cache: `localStorage.removeItem('site_events')`
    - Ensure `getEvents()` function is not throwing errors
  </Accordion>

  <Accordion title="Performance data missing">
    Performance metrics use the Performance API:
    - Ensure browser supports `performance.getEntriesByType()`
    - Check that tracker is initialized on page load
    - Some metrics only appear after user interaction
    - Private/incognito mode may limit Performance API access
  </Accordion>

  <Accordion title="Cannot access users tab">
    If you can't see the Users section:
    - Verify you're logged in as SUPER_ADMIN
    - Check `localStorage.getItem('user_session')`
    - Ensure role is exactly "SUPER_ADMIN" (case-sensitive)
    - Try logging out and back in to refresh session
  </Accordion>
</AccordionGroup>

## Security Considerations

<Warning>
**Critical**: User data stored in localStorage is not encrypted and can be accessed via browser DevTools. For production systems:

- Store only non-sensitive data in localStorage
- Use backend authentication for admin access
- Implement rate limiting on user data endpoints
- Encrypt sensitive fields before localStorage storage
- Use HTTPS for all admin panel access
- Implement session timeout for admin users
- Log all admin actions for security auditing
</Warning>
