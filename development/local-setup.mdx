# Local Development Setup

This guide will walk you through setting up the CONIITI 2026 platform on your local machine for development.

## Prerequisites

Before you begin, ensure you have the following installed:

- **Node.js** v16 or higher
- **npm** (comes with Node.js)
- **Git** for version control
- A code editor (VS Code recommended)
- A Supabase account (for database and authentication)

## Project Structure

The CONIITI 2026 platform is a monorepo containing:

```
coniiti-2026/
├── backend/          # Express.js API server
├── frontend/         # React + Vite application
│   └── vite-project/
├── docs/            # Documentation
└── package.json     # Root package with dev scripts
```

## Quick Start

<Steps>
  <Step title="Clone the Repository">
    Clone the project repository to your local machine:
    
    ```bash
    git clone <repository-url>
    cd coniiti-2026
    ```
  </Step>

  <Step title="Install Dependencies">
    Install dependencies for both frontend and backend:
    
    <CodeGroup>
    ```bash Root (Installs all)
    npm install
    ```
    
    ```bash Backend Only
    cd backend
    npm install
    ```
    
    ```bash Frontend Only
    cd frontend/vite-project
    npm install
    ```
    </CodeGroup>
  </Step>

  <Step title="Configure Environment Variables">
    Set up your environment variables for both frontend and backend. See the [Environment Variables](/development/environment-variables) guide for detailed configuration.
    
    **Backend** - Create `backend/.env`:
    ```bash
    SUPABASE_URL=your_supabase_url
    SUPABASE_ANON_KEY=your_anon_key
    SUPABASE_SERVICE_ROLE_KEY=your_service_role_key
    PORT=3000
    ```
    
    **Frontend** - Create `frontend/vite-project/.env`:
    ```bash
    VITE_SUPABASE_URL=your_supabase_url
    VITE_SUPABASE_ANON_KEY=your_anon_key
    VITE_API_URL=http://localhost:3000/api
    ```
  </Step>

  <Step title="Set Up Database">
    Configure your Supabase database. See the [Database Setup](/development/database-setup) guide for detailed instructions.
  </Step>

  <Step title="Start Development Servers">
    You can run both servers simultaneously or separately:
    
    <CodeGroup>
    ```bash Both Servers (Recommended)
    # From the root directory
    npm run dev
    ```
    
    ```bash Backend Only
    cd backend
    npm run dev
    ```
    
    ```bash Frontend Only
    cd frontend/vite-project
    npm run dev
    ```
    </CodeGroup>
    
    The servers will start on:
    - **Frontend**: http://localhost:5173
    - **Backend API**: http://localhost:3000
  </Step>
</Steps>

## Development Workflow

### Running the Backend

The backend is an Express.js API server with TypeScript:

```bash
cd backend
npm run dev    # Starts with nodemon (auto-reload)
npm run build  # Compiles TypeScript
npm start      # Runs compiled JavaScript
```

**Backend Structure:**
```
backend/
├── src/
│   ├── app.ts                    # Express app configuration
│   ├── server.ts                 # Server entry point
│   ├── config/
│   │   └── supabase.ts          # Supabase client setup
│   ├── middleware/
│   │   └── auth.middleware.ts   # Authentication middleware
│   └── modules/
│       ├── evento/              # Events management
│       ├── ponencia/            # Presentations management
│       ├── ponente/             # Speakers management
│       ├── inscripcion/         # Registrations management
│       └── usuario/             # User management
└── package.json
```

**API Endpoints:**
- `/api/usuarios` - User management and authentication
- `/api/eventos` - Event operations
- `/api/ponencias` - Conference presentations
- `/api/ponentes` - Speaker management
- `/api/inscripciones` - User registrations

### Running the Frontend

The frontend is built with React 19, TypeScript, and Vite:

```bash
cd frontend/vite-project
npm run dev      # Starts Vite dev server
npm run build    # Production build
npm run preview  # Preview production build
```

**Key Technologies:**
- **React 19** with TypeScript
- **Vite 7** for fast builds and HMR
- **React Router v7** for navigation
- **Tailwind CSS v4** for styling
- **Supabase** for authentication
- **Lucide React** for icons
- **Recharts** for data visualization

## Common Issues

<AccordionGroup>
  <Accordion title="Port Already in Use">
    If port 3000 or 5173 is already in use:
    
    **Backend**: Change the `PORT` in `backend/.env`
    
    **Frontend**: Vite will automatically try the next available port, or you can specify one in `vite.config.ts`:
    ```typescript
    export default defineConfig({
      server: {
        port: 5174
      }
    })
    ```
  </Accordion>

  <Accordion title="Module Not Found Errors">
    If you encounter module errors:
    
    ```bash
    # Clear node_modules and reinstall
    rm -rf node_modules package-lock.json
    npm install
    ```
  </Accordion>

  <Accordion title="Supabase Connection Errors">
    Verify your Supabase credentials:
    - Check that `.env` files exist in both frontend and backend
    - Ensure `SUPABASE_URL` and `SUPABASE_ANON_KEY` are correct
    - Test your credentials in the Supabase dashboard
    - Check that your Supabase project is active
  </Accordion>

  <Accordion title="CORS Errors">
    If you see CORS errors in the browser console:
    - Ensure the backend is running
    - Verify `VITE_API_URL` points to `http://localhost:3000/api`
    - Check that CORS is enabled in `backend/src/app.ts`
  </Accordion>

  <Accordion title="TypeScript Compilation Errors">
    If TypeScript won't compile:
    
    ```bash
    # Clear TypeScript cache
    rm -rf dist
    npm run build
    ```
  </Accordion>
</AccordionGroup>

## Development Tips

- Use the browser's React DevTools for debugging components
- Check the Network tab to debug API calls
- Backend logs appear in the terminal where you ran `npm run dev`
- Frontend errors appear in both the browser console and terminal
- Use `console.log()` sparingly; prefer debugging tools
- Enable Supabase local development for offline work
