## Overview

The CONIITI 2026 backend is built with **Express.js** and **TypeScript**, following a modular architecture pattern. The project emphasizes separation of concerns with dedicated directories for configuration, middleware, and feature modules.

## Directory Structure

```
src/
├── app.ts                    # Express app configuration
├── server.ts                 # Server entry point
├── config/                   # Configuration files
│   └── supabase.ts          # Supabase client setup
├── middleware/               # Express middleware
│   └── auth.middleware.ts   # Authentication middleware
└── modules/                  # Feature modules
    ├── evento/              # Event management
    ├── inscripcion/         # Registration management
    ├── ponencia/            # Presentation management
    ├── ponente/             # Speaker management
    └── usuario/             # User management
```

## Core Files

### app.ts

The main Express application configuration file. It sets up middleware and registers all module routes.

**Source:** `src/app.ts`

```typescript
import express from 'express'
import cors from 'cors'

import usuarioRoutes from './modules/usuario/usuario.routes'
import eventoRoutes from './modules/evento/evento.routes'
import ponencias from './modules/ponencia/ponencia.routes'
import inscripciones from './modules/inscripcion/inscripcion.routes'
import ponentes from './modules/ponente/ponente.routes'

const app = express()

// Middleware
app.use(cors())
app.use(express.json())

// Route registration
app.use('/api/usuarios', usuarioRoutes)
app.use('/api/eventos', eventoRoutes)
app.use('/api/ponencias', ponencias)
app.use('/api/inscripciones', inscripciones)
app.use('/api/ponentes', ponentes)

export default app
```

**Key responsibilities:**
- Configure CORS for cross-origin requests
- Parse JSON request bodies
- Register module routes under `/api/*` prefix
- Export configured Express app

### server.ts

The application entry point that starts the HTTP server.

**Source:** `src/server.ts`

```typescript
import 'dotenv/config'
import app from './app'

const PORT = process.env.PORT || 3000

app.listen(PORT, () => {
  console.log(`Servidor corriendo en: http://localhost:${PORT}`)
})
```

**Key responsibilities:**
- Load environment variables via dotenv
- Start the Express server on configured port
- Provide startup confirmation logging

## Modules Directory

The `modules/` directory contains feature-based modules. Each module follows a consistent structure:

- **routes.ts** - Express router and endpoint definitions
- **controller.ts** - Request/response handlers
- **service.ts** - Business logic and database operations
- **dto.ts** - Data transfer objects and validation schemas (optional)

### Available Modules

| Module | Description | Endpoint |
|--------|-------------|----------|

## Architecture Principles

### Separation of Concerns

The architecture strictly separates:
- **Routes** - HTTP endpoint definitions
- **Controllers** - Request/response handling
- **Services** - Business logic and data access
- **Middleware** - Cross-cutting concerns (auth, logging)
- **Config** - Application configuration

### Modular Design

Each feature is self-contained in its own module directory, making the codebase:
- Easy to navigate
- Simple to test in isolation
- Straightforward to extend with new features
- Clear ownership of responsibilities

### Type Safety

TypeScript provides compile-time type checking across:
- Request/response objects
- Database schemas
- Service interfaces
- Configuration objects
