---
title: "List Speakers"
api: "GET /api/ponentes"
description: "Retrieve a list of all speakers registered in the CONIITI 2026 platform"
---

## Endpoint

```
GET /api/ponentes
```

## Authentication

This endpoint is **public** and does not require authentication.

## Response

Returns an array of speaker objects.

<ResponseField name="speakers" type="array">
  Array of speaker objects
  
  <Expandable title="Speaker Object">
    <ResponseField name="id" type="string">
      Unique identifier for the speaker
    </ResponseField>
    
    <ResponseField name="nombre" type="string">
      Full name of the speaker
    </ResponseField>
    
    <ResponseField name="biografia" type="string">
      Speaker biography
    </ResponseField>
    
    <ResponseField name="foto" type="string">
      Base64-encoded photo or URL to speaker photo
    </ResponseField>
    
    <ResponseField name="cargo" type="string">
      Position or title of the speaker
    </ResponseField>
    
    <ResponseField name="institucion" type="string">
      Institution or organization the speaker represents
    </ResponseField>
    
    <ResponseField name="created_at" type="string">
      Timestamp of when the speaker was created
    </ResponseField>
  </Expandable>
</ResponseField>

## Example Request

<CodeGroup>
```bash cURL
curl https://your-api-url.com/api/ponentes
```

```javascript JavaScript
const response = await fetch('https://your-api-url.com/api/ponentes');
const speakers = await response.json();
console.log(speakers);
```

```python Python
import requests

response = requests.get('https://your-api-url.com/api/ponentes')
speakers = response.json()
print(speakers)
```
</CodeGroup>

## Status Codes

<ResponseField name="200" type="Success">
  Successfully retrieved the list of speakers
</ResponseField>

<ResponseField name="400" type="Error">
  Bad request - Database error occurred
  
  ```json
  {
    "error": "Database query failed"
  }
  ```
</ResponseField>

## Notes

- The `foto` field typically contains a base64-encoded image string
- All speakers are returned regardless of their association with conferences
- Results are not paginated in the current implementation
- To get speakers for a specific conference, use the conference endpoints which include speaker details
