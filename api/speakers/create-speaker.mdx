## Endpoint

```
POST /api/ponentes
```

## Authentication

This endpoint requires authentication. Include a valid JWT token in the Authorization header.

```
Authorization: Bearer YOUR_JWT_TOKEN
```

## Request Body

<ParamField path="nombre" type="string" required>
  Full name of the speaker
</ParamField>

<ParamField path="biografia" type="string" required>
  Biography or description of the speaker's background and expertise
</ParamField>

<ParamField path="foto" type="string" optional>
  Base64-encoded photo or URL to speaker photo. If not provided, a default avatar may be used.
</ParamField>

<ParamField path="cargo" type="string" required>
  Position or title of the speaker (e.g., "Director de Investigación", "Profesor Titular")
</ParamField>

<ParamField path="institucion" type="string" required>
  Institution or organization the speaker represents
</ParamField>

## Response

Returns the newly created speaker object.

<ResponseField name="id" type="string">
  Unique identifier for the newly created speaker
</ResponseField>

<ResponseField name="nombre" type="string">
  Full name of the speaker
</ResponseField>

<ResponseField name="biografia" type="string">
  Speaker biography
</ResponseField>

<ResponseField name="foto" type="string">
  Base64-encoded photo or URL to speaker photo
</ResponseField>

<ResponseField name="cargo" type="string">
  Position or title of the speaker
</ResponseField>

<ResponseField name="institucion" type="string">
  Institution or organization
</ResponseField>

<ResponseField name="created_at" type="string">
  Timestamp of when the speaker was created
</ResponseField>

## Example Request

<CodeGroup>
```bash cURL
curl -X POST https://your-api-url.com/api/ponentes \
  -H "Authorization: Bearer YOUR_JWT_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "nombre": "Dr. María González",
    "biografia": "Experta en inteligencia artificial con 15 años de experiencia en investigación y desarrollo de sistemas inteligentes.",
    "foto": "data:image/jpeg;base64,/9j/4AAQSkZJRg...",
    "cargo": "Directora de Investigación",
    "institucion": "Universidad Católica de Colombia"
  }'
```

```javascript JavaScript
const response = await fetch('https://your-api-url.com/api/ponentes', {
  method: 'POST',
  headers: {
    'Authorization': 'Bearer YOUR_JWT_TOKEN',
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    nombre: "Dr. María González",
    biografia: "Experta en inteligencia artificial con 15 años de experiencia",
    foto: "data:image/jpeg;base64,/9j/4AAQSkZJRg...",
    cargo: "Directora de Investigación",
    institucion: "Universidad Católica de Colombia"
  })
});

const speaker = await response.json();
console.log(speaker);
```

```python Python
import requests

headers = {
    'Authorization': 'Bearer YOUR_JWT_TOKEN',
    'Content-Type': 'application/json'
}

data = {
    "nombre": "Dr. María González",
    "biografia": "Experta en inteligencia artificial",
    "foto": "data:image/jpeg;base64,/9j/4AAQSkZJRg...",
    "cargo": "Directora de Investigación",
    "institucion": "Universidad Católica de Colombia"
}

response = requests.post(
    'https://your-api-url.com/api/ponentes',
    headers=headers,
    json=data
)

speaker = response.json()
print(speaker)
```
</CodeGroup>

## Status Codes

<ResponseField name="201" type="Created">
  Speaker successfully created
</ResponseField>

<ResponseField name="400" type="Error">
  Bad request - Invalid data provided
  
  ```json
  {
    "error": "Missing required field: nombre"
  }
  ```
</ResponseField>

<ResponseField name="401" type="Error">
  Unauthorized - Invalid or missing authentication token
  
  ```json
  {
    "error": "No se proporcionó token de autorización"
  }
  ```
</ResponseField>

<ResponseField name="403" type="Error">
  Forbidden - Database insert failed
  
  ```json
  {
    "error": "Failed to create speaker"
  }
  ```
</ResponseField>

## Notes

<Note>
  The `foto` field accepts base64-encoded images. For best results:
  - Use JPEG or PNG format
  - Keep file size under 1MB for optimal performance
  - Recommended dimensions: 400x400 pixels
</Note>

<Warning>
  This endpoint requires authentication. Users must be logged in and have a valid session to create speakers.
</Warning>
