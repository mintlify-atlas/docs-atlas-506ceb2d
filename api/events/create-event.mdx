## Endpoint

```
POST /api/eventos
```

## Authentication

This endpoint **requires authentication**. You must include a valid authentication token in the request headers.

**Headers:**
- `Authorization: Bearer <token>`

## Request Body

<ParamField body="nombre" type="string" required>
  Name of the event
</ParamField>

<ParamField body="descripcion" type="string">
  Description of the event
</ParamField>

<ParamField body="fecha_inicio" type="string" required>
  Start date and time of the event (ISO 8601 format)
</ParamField>

<ParamField body="fecha_fin" type="string" required>
  End date and time of the event (ISO 8601 format)
</ParamField>

<ParamField body="ubicacion" type="string">
  Location where the event will take place
</ParamField>

<ParamField body="capacidad" type="integer">
  Maximum capacity for the event
</ParamField>

  <ResponseField name="nombre" type="string">

## Examples

### Request

```bash
curl -X POST https://api.coniiti2026.com/api/eventos \
  -H "Authorization: Bearer YOUR_TOKEN_HERE" \
  -H "Content-Type: application/json" \
  -d '{
    "nombre": "Hackathon CONIITI",
    "descripcion": "Competencia de programación de 24 horas",
    "fecha_inicio": "2026-03-16T08:00:00Z",
    "fecha_fin": "2026-03-17T08:00:00Z",
    "ubicacion": "Pabellón de Innovación",
    "capacidad": 100
  }'
```

## Error Responses

<ResponseField name="error" type="string">
  Error message describing what went wrong
</ResponseField>

| Status Code | Description |
|------------|-------------|
| 201 | Event created successfully |
| 401 | Unauthorized - Invalid or missing authentication token |
| 403 | Forbidden - User doesn't have permission to create events |

## Notes

- The authentication middleware validates the JWT token before allowing event creation
- All date fields should be in ISO 8601 format (YYYY-MM-DDTHH:mm:ssZ)
- The `fecha_fin` should be after `fecha_inicio`
- Event capacity should be a positive integer
