## Overview

The profile endpoints allow authenticated users to view, create, and update their profile information in the CONIITI 2026 system.

<Warning>
All profile endpoints require authentication. Include a valid Bearer token in the Authorization header.
</Warning>

---

## Get Profile

```
GET /perfil
```

Retrieves the authenticated user's profile information.

### Authentication

<ParamField header="Authorization" type="string" required>
  Bearer token for authentication
  
  ```
  Authorization: Bearer <your-access-token>
  ```
</ParamField>

### Response

<ResponseField name="id" type="string">
  The user's unique ID (UUID)
</ResponseField>

<ResponseField name="nombre_completo" type="string">
  The user's full name
</ResponseField>

<ResponseField name="email" type="string">
  The user's email address
</ResponseField>

<ResponseField name="rol" type="string">
  The user's role (e.g., ESTUDIANTE, USER)
</ResponseField>

<ResponseField name="carrera" type="string">
  The user's academic career or program
</ResponseField>

<ResponseField name="genero" type="string">
  The user's gender
</ResponseField>

<ResponseField name="documento" type="string">
  The user's identification document number
</ResponseField>

<ResponseField name="codigo_institucional" type="string">
  The user's institutional code (for students)
</ResponseField>

<ResponseField name="created_at" type="string">
  Timestamp of profile creation
</ResponseField>

### Examples

#### Request

```bash cURL
curl -X GET https://api.coniiti2026.com/perfil \
  -H "Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
```

#### Success Response (200)

```json
{
  "id": "a1b2c3d4-e5f6-7890-abcd-ef1234567890",
  "nombre_completo": "Juan Pérez García",
  "email": "juan.perez@universidad.edu",
  "rol": "ESTUDIANTE",
  "carrera": "Ingeniería de Sistemas",
  "genero": "Masculino",
  "documento": "12345678",
  "codigo_institucional": "2021100123",
  "created_at": "2026-01-15T10:30:00Z"
}
```

#### Error Response (400)

```json
{
  "error": "Profile not found"
}
```

#### Unauthorized (401)

```json
{
  "error": "Token requerido"
}
```

or

```json
{
  "error": "Token inválido"
}
```

## Create Profile

```
POST /perfil
```

Creates a profile for the authenticated user. This is typically used if a profile wasn't created during registration.

### Authentication

<ParamField header="Authorization" type="string" required>
  Bearer token for authentication
  
  ```
  Authorization: Bearer <your-access-token>
  ```
</ParamField>

### Request Body

<ParamField body="nombre_completo" type="string" required>
  The user's full name
</ParamField>

<ParamField body="rol" type="string" default="USER">
  The user's role in the system
</ParamField>

### Response

<ResponseField name="message" type="string">
  Success message confirming profile creation
</ResponseField>

### Examples

#### Request

```bash cURL
curl -X POST https://api.coniiti2026.com/perfil \
  -H "Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..." \
  -H "Content-Type: application/json" \
  -d '{
    "nombre_completo": "María González López",
    "rol": "USER"
  }'
```

#### Success Response (201)

```json
{
  "message": "Perfil creado correctamente"
}
```

#### Error Response (400)

```json
{
  "error": "duplicate key value violates unique constraint"
}
```

#### Unauthorized (401)

```json
{
  "error": "Token requerido"
}
```

## Update Profile

```
PUT /perfil
```

Updates the authenticated user's profile information.

### Authentication

<ParamField header="Authorization" type="string" required>
  Bearer token for authentication
  
  ```
  Authorization: Bearer <your-access-token>
  ```
</ParamField>

### Request Body

<ParamField body="nombre_completo" type="string">
  Updated full name
</ParamField>

<ParamField body="carrera" type="string">
  Updated academic career or program
</ParamField>

<Note>
Only include the fields you want to update. Other fields will remain unchanged.
</Note>

### Response

<ResponseField name="message" type="string">
  Success message confirming profile update
</ResponseField>

### Examples

#### Request

```bash cURL
curl -X PUT https://api.coniiti2026.com/perfil \
  -H "Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..." \
  -H "Content-Type: application/json" \
  -d '{
    "nombre_completo": "Juan Pérez García",
    "carrera": "Ingeniería de Software"
  }'
```

#### Success Response (200)

```json
{
  "message": "Perfil actualizado correctamente"
}
```

#### Error Response (400)

```json
{
  "error": "No rows updated"
}
```

#### Unauthorized (401)

```json
{
  "error": "Token requerido"
}
```

## Common Error Responses

All profile endpoints may return the following authentication errors:

### Missing Token (401)

```json
{
  "error": "Token requerido"
}
```

Occurs when the Authorization header is missing.

### Invalid Token (401)

```json
{
  "error": "Token inválido"
}
```

Occurs when the provided token is expired or invalid.

### Unauthorized (401)

```json
{
  "error": "No autorizado"
}
```

Occurs when token verification fails for any other reason.

## Implementation Notes

- All profile operations use the `perfil_usuario` table
- The user ID is automatically extracted from the JWT token
- Profile updates are partial - only specified fields are updated
- The system uses Supabase for authentication and database operations
- Tokens are validated on every request using the auth middleware
